This is a walkthrough of the concept-model idiom. This idioms provides unified
interface as a **concept**, while users can plug in different **model** that
fulfills the interface. Models don't need to inherit from the concept.

By going through the commits, you can see modifications that expands the 
document object support random type objects(model) as long as they fulfills
the concept(draw, copy).
